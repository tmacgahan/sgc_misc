9.10a. Data import and exportUser friendly and practical Data Import and Export (DI/E) designs are drilled down to User Interface (UI) and capability of Data Analytical Processing (DAP).  The purpose of the UI is to let users capable to operating DI/E intuitively without necessity of guidance. A good example can be that two buttons which respectively represent Import and Export in main UI toolbar or dropdown menu. For DAP, the standard file formats such as .csv, .xls and .doc can be easily parsed, analyzed and exported in the backend system regardless of the programming languages, with aid of existing open source APIs or self-coding. No special equipment or software is needed for achieving the business goals. The technical implementation is fairly straightforward.
